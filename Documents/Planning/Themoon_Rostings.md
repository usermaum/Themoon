# 더문(The Moon) 로스팅 및 재고 관리 운영 계획안

## 1. 개요 (Overview)
본 문서는 '더문(The Moon)'의 효율적인 로스팅 생산 관리 및 재고 운영을 위한 기준 정보와 업무 시나리오를 정의합니다. 원두의 입고부터 로스팅, 블렌딩, 그리고 최종 제품 생산에 이르는 프로세스를 체계화하여 데이터 기반의 운영을 목표로 합니다.

### 1.1 용어 정의 (Terminology)

**재료 및 제품 분류**:
*   **생두 (Green Bean)**: 로스팅 **전**의 커피 원두 (입고된 원재료)
*   **원두 (Roasted Bean)**: 로스팅 **후**의 커피 원두
    *   **싱글 오리진 원두**: 한 종류의 생두만 로스팅한 원두
    *   **블렌드 원두**: 여러 종류의 생두를 배합하여 함께 로스팅한 원두

**로스팅 프로필 (Roasting Profile)**:
*   **신콩 (Light Roast)**: 밝은 로스팅 (약볶음) - 산미와 풍미 강조
*   **탄콩 (Dark Roast)**: 깊은 로스팅 (강볶음) - 바디감과 쓴맛 강조

**재고 관리 원칙**:
*   **생두**: 품종별로 재고 관리 (예: 예가체프 생두)
*   **싱글 오리진 원두**: **품종 × 로스팅 프로필**로 분류 관리
    *   예: "예가체프-신콩", "예가체프-탄콩"은 **별도의 재고 항목**
*   **블렌드 원두**: 블렌드명으로 관리 (예: 풀문, 뉴문)

## 3. 운영 시나리오 (Operational Scenarios)

실제 매장 운영 시 발생하는 주요 업무 흐름을 정의합니다.

### 시나리오 1: 자재 입고 (Material Inbound)
외부 업체로부터 생두를 구매하고, 거래 명세서(견적서, 영수증)를 기반으로 입고를 처리하는 단계입니다.

1.  **문서 확보**: 거래처로부터 받은 견적서 또는 영수증(이미지/PDF)을 준비합니다.
2.  **입고 등록 (OCR)**:
    *   시스템에 문서를 업로드합니다.
    *   시스템이 문서 내용을 자동 인식하여 **품목명(원두)**, **수량(kg)**, **단가(원)** 정보를 추출합니다.
3.  **데이터 검증**:
    *   추출된 데이터와 실제 입고된 물품이 일치하는지 확인합니다.
    *   인식 오류가 있거나 매핑되지 않은 원두가 있다면 수정합니다.
4.  **입고 확정**:
    *   검증이 완료되면 입고 처리를 완료합니다.
    *   **생두 재고**가 증가하고, 해당 회차의 **매입 단가**가 평균 단가에 반영됩니다.

### 시나리오 2: 싱글 오리진 로스팅 생산 (Single Origin Roasting Process)
한 종류의 생두를 볶아 싱글 오리진 원두로 가공하는 단계입니다.

#### 2.1 작업 준비
*   로스팅할 **생두 품목**을 선택합니다. (예: 예가체프 생두)
*   **로스팅 프로필**을 선택합니다: **신콩** 또는 **탄콩**
*   목표 원두 생산량(kg)을 입력합니다.

#### 2.2 생두 투입량 계산 (자동)
시스템은 목표 생산량과 해당 생두의 **평균 손실률** (또는 설정된 기본 손실률)을 기반으로 필요한 생두량을 자동 계산합니다.

**계산 공식**:
```
필요 생두량 = 목표 원두 생산량 / (1 - 손실률)
```

**예시**:
*   목표: 예가체프-신콩 원두 10kg 생산
*   평균 손실률: 15%
*   필요 생두량: 10kg / (1 - 0.15) = **11.76kg**

#### 2.3 로스팅 수행 및 결과 기록
*   계산된 생두량(또는 사용자가 수정한 양)을 투입하여 로스팅을 진행합니다.
*   선택한 프로필(신콩/탄콩)에 따라 로스팅 온도/시간을 조정합니다.
*   로스팅 완료 후 **실제 생산된 원두량**을 측정하여 기록합니다.

#### 2.4 데이터 처리
1.  **손실률 계산 (자동)**:
    ```
    실제 손실률(%) = (생두 투입량 - 원두 생산량) / 생두 투입량 × 100
    ```
    *   예: 11.76kg 투입 → 10kg 생산 → 손실률 = (11.76 - 10) / 11.76 × 100 = **14.97%**

2.  **재고 반영**:
    *   **생두 재고**: 예가체프 생두 **-11.76kg**
    *   **원두 재고**: 예가체프-**신콩** 원두 **+10kg** (프로필별로 분리된 재고)
    
    > **중요**: 동일한 예가체프 생두를 사용하더라도, "예가체프-신콩"과 "예가체프-탄콩"은 **별도의 재고 항목**으로 관리됩니다.

3.  **원가 계산**:
    ```
    원두 단가 = (생두 매입 단가 × 투입량) / 생산량
    ```
    *   예: 생두 단가 14,000원/kg, 투입 11.76kg, 생산 10kg
    *   원두 단가 = (14,000 × 11.76) / 10 = **16,464원/kg**
    *   이 원가는 "예가체프-신콩" 원두에 적용됩니다.

4.  **프로필별 손실률 추적**:
    *   시스템은 품종 × 프로필 조합별로 손실률 이력을 기록합니다.
    *   예: "예가체프-신콩" 평균 손실률 15.2%, "예가체프-탄콩" 평균 손실률 16.8%

### 시나리오 3: 블렌딩 제조 (Blending Process)
**여러 종류의 생두를 배합하여 로스팅하는 방식 (Pre-roast Blending)**

#### 3.1 제품 선택 및 목표 설정
*   생산할 블렌드 종류(예: 풀문)를 선택합니다.
*   목표 블렌드 원두 생산량(kg)을 입력합니다.

#### 3.2 생두 배합 산출 (자동)
시스템은 표준 레시피, 목표 생산량, 그리고 **블렌드의 예상 손실률**을 기반으로 필요한 각 생두의 양을 계산합니다.

**계산 공식**:
```
총 필요 생두량 = 목표 블렌드 원두 생산량 / (1 - 블렌드 손실률)
각 원두별 필요 생두량 = 총 필요 생두량 × 해당 원두 배합 비율
```

**예시: 풀문 10kg 생산 시**
*   레시피: 마사이 40%, 안티구아 40%, 모모라 10%, 시다모 G4 10%
*   블렌드 예상 손실률: 15%

**단계 1**: 총 필요 생두량 계산
```
총 필요 생두량 = 10kg / (1 - 0.15) = 11.76kg
```

**단계 2**: 각 원두별 필요 생두량 계산
*   마사이 생두: 11.76kg × 40% = **4.71kg**
*   안티구아 생두: 11.76kg × 40% = **4.71kg**
*   모모라 생두: 11.76kg × 10% = **1.18kg**
*   시다모 G4 생두: 11.76kg × 10% = **1.18kg**

#### 3.3 생두 배합 및 로스팅 수행
1.  계산된 양만큼 각 생두를 계량합니다.
2.  생두들을 혼합합니다.
3.  혼합된 생두를 **함께 로스팅**합니다.
4.  로스팅 완료 후 **실제 생산된 블렌드 원두량**을 측정하여 기록합니다.

#### 3.4 데이터 처리
1.  **손실률 계산 (자동)**:
    ```
    실제 손실률(%) = (총 생두 투입량 - 블렌드 원두 생산량) / 총 생두 투입량 × 100
    ```
    *   예: 11.76kg 생두 혼합 투입 → 10kg 블렌드 원두 생산
    *   손실률 = (11.76 - 10) / 11.76 × 100 = **14.97%**

2.  **재고 반영**:
    *   마사이 생두: **-4.71kg**
    *   안티구아 생두: **-4.71kg**
    *   모모라 생두: **-1.18kg**
    *   시다모 G4 생두: **-1.18kg**
    *   풀문 블렌드 원두: **+10kg** (실제 생산량)

3.  **원가 계산 (가중평균)**:
    ```
    블렌드 원두 단가 = Σ(각 생두 단가 × 투입량) / 블렌드 원두 생산량
    ```
    *   예: 
        - 마사이 생두 14,000원/kg × 4.71kg = 65,940원
        - 안티구아 생두 16,000원/kg × 4.71kg = 75,360원
        - 모모라 생두 22,000원/kg × 1.18kg = 25,960원
        - 시다모 생두 12,000원/kg × 1.18kg = 14,160원
    *   총 생두 비용 = 181,420원
    *   **블렌드 원두 단가 = 181,420원 / 10kg = 18,142원/kg**

지속적인 수익성 확인과 적정 재고 유지를 위한 관리 프로세스입니다.

1.  **원가 산정**:

## 5. 개선 및 검토 사항 (Improvement Recommendations)

### 5.1 프로세스 명확화
1.  **로스팅 프로필 관리 체계화**
    *   현재 상태: 신콩/탄콩 선택 기능은 정의되었으나, 구체적인 로스팅 파라미터 기준이 없음
    *   **필요 작업**:
        *   **신콩 프로필**: 권장 온도 범위, 로스팅 시간, 1차 크랙 시점 등 정의
        *   **탄콩 프로필**: 권장 온도 범위, 로스팅 시간, 2차 크랙 진행도 등 정의
        *   품종별 프로필 조합 매뉴얼 작성 (예: 예가체프는 신콩 권장, 브라질은 탄콩 적합 등)
        *   각 프로필별 평균 손실률 데이터 축적 및 비교 분석

2.  **블렌드 vs 싱글 오리진 명확한 구분**
    *   **싱글 오리진**: 한 종류의 생두 → 로스팅 → 신콩/탄콩으로 분류된 원두
    *   **블렌드**: 여러 생두 배합 → **함께** 로스팅 → 블렌드 원두 (신콩/탄콩 구분 없음)
    *   블렌드는 Pre-roast blending 방식이므로 15% 손실률이 정상적임

3.  **재고 관리 체계 정교화**
    *   **SKU 체계**:
        *   생두: `{품종명}` (예: "예가체프")
        *   싱글 원두: `{품종명}-{프로필}` (예: "예가체프-신콩", "예가체프-탄콩")
        *   블렌드 원두: `{블렌드명}` (예: "풀문", "뉴문")
    *   재고 이동 추적: 생두 → 싱글 원두(신콩/탄콩) → 판매 또는 블렌드 재료로 사용 (Post-blend의 경우)


### 5.2 데이터 관리
1.  **손실률 추적 및 분석**
    *   각 로스팅 작업마다 실제 손실률을 기록하고 평균값을 자동 업데이트
    *   원두별/프로필별 손실률 패턴 분석 기능 추가
    *   이상치(outlier) 발생 시 알림 기능

2.  **원두 이름 표준화**
    *   현재: "AA FAQ (마사이)", "케냐 AA FAQ" 등 동일 원두에 대한 표기 불일치
    *   **권장**: 마스터 데이터와 입고 데이터 간 원두명 매핑 테이블 관리

3.  **단가 변동 추적**
    *   브라질: 9,900원(2024.12) → 13,100원(2025.03) (32% 상승)
    *   **권장**: 단가 변동 이력 그래프 및 구매 시기 최적화 분석 도구

### 5.3 시스템 기능 제안
1.  **자동 발주 제안**
    *   안전 재고 임계값 설정
    *   소비 패턴 기반 자동 발주량 계산
    *   선호 공급자 및 최저가 정보 연동

2.  **수익성 분석 대시보드**
    *   블렌드별 원가율 실시간 모니터링
    *   판매가 대비 마진율 표시
    *   권장 판매가 자동 계산(목표 마진율 기반)

3.  **로스팅 기록 상세화**
    *   로스팅 일시, 담당자, 기계 온도 프로파일
    *   품질 평가 점수 (컵핑 노트)
    *   배치 번호(Lot Number) 추적 시스템
